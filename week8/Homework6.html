Homework 6

Team Assignment

By the lab next week please have:

  1. Running code for "vote for projects" use case.
    • display something to vote on with choices, and accept a vote.
    • have a desktop GUI or web-based UI.
    • app knows who is voting, even if you don't use authentication.
    • records vote in a log4j log file (at least).
  2. Software level sequence diagram that shows how your software handles this use case, and class diagram that shows the structure.
  3. Code must be checked in to VCS.
  4. Assign each person on your team the task of writing JUnit 4 tests for at least one class (different class for each person) that the person is writing.

Individual Assignment

Write JUnit 4 tests for at least one class you wrote for the team assignment. Write meaningful tests for behavior, not trivial tests (testing simple get/set methods are trivial and usually omitted). Write tests for desired behavior, especially borderline cases, rather than methods.

Consider using Test Driven Development (Larman Chapter 21, or search the web). In TDD, you decide on the desired behavior first (including things you don't want to happen) and write unit tests for it. At first, the tests fail or won't compile because you haven't written the code to test yet. Write stubs ("do nothing" methods) for the code to test and start writing real methods -- testing as you code.
